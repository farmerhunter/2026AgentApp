# 设计简报

项目：面向初中学习的 AI 学习智能体

报名方向：2026 北京青少年人工智能应用实践活动 - AI 智能体应用

## 1. 项目定位

本项目演示一个主动式学习智能体 Hermes。它面向初中学生的日常学习场景，帮助学生把试卷、错题本、作业、笔记和学习问题转化为可执行的复习行动。

第一版不是完整的生产级教育平台，而是一个可演示、可解释、能体现智能体价值的比赛作品。

核心价值：

- 从学生上传的学习材料中提取有用信息。
- 通过人机协作确认真正需要记录的重点题。
- 由 Hermes 总结错因、知识点、复习建议和下周行动。
- 在 Web 前端中展示周报、重点题和学习建议。

## 2. 目标用户

- 学生：上传试卷或错题本，确认需要记录的重点题，查看复习建议。
- 家长：了解本周学习情况、薄弱点和下一步计划。
- 老师：参考学生错题和知识点分布，了解学习进展。
- 比赛评委：观察 AI 智能体如何完成材料处理、分析、总结和行动建议闭环。

## 3. 第一版核心体验

第一版演示的主流程：

```text
上传试卷/错题本图像
  -> 腾讯云 QuestionSplitOCR 试卷切题
  -> Hermes Web 展示题目框和切题列表
  -> 学生手动选择需要记录的重点题
  -> 学生补充得分、知识点、错因和备注
  -> Hermes 生成重点题记录和学习建议
  -> Hermes 生成周报 JSON
  -> Web 前端展示历史周报和行动建议
```

这个流程的重点不是追求全自动批改，而是展示智能体如何把学生已有学习材料组织成可跟进的行动。

## 4. 演示重点

- 主动分析：上传材料后，Hermes 能生成下一步分析和建议。
- 人机协作：腾讯云负责试卷切题，学生负责确认重点题和语义信息。
- 可解释输出：每个建议应能追溯到上传资料、重点题或人工确认记录。
- 周报展示：Web 前端展示本周总结、上传材料、重点题、学习建议和下周行动。
- 可演进架构：演示阶段用静态 JSON，后续可迁移到数据库和 REST API。

## 5. 第一版不做的内容

为保证演示可落地，第一版不承诺：

- 自动可靠识别老师红笔对勾、叉号、圈画和划线。
- 自动理解老师手写批注。
- 自动判断老师给分、扣分和扣分原因。
- 自动为全部题目生成准确知识点和错因。
- 支持多学生、多班级、权限系统和长期生产级数据治理。

这些能力可以作为后续阶段研究方向，但不应成为第一版演示成败的关键依赖。

## 6. Web 体验原则

Hermes Web 第一版应是工作台，而不是营销首页。

设计原则：

- 第一屏直接进入可操作的学习助手界面。
- 页面围绕上传试卷、确认重点题、查看周报三个任务组织。
- 视觉风格清爽、可信、专业，适合教育科技场景。
- 信息密度适中，重点突出，不使用夸张装饰和营销式排版。
- 中文内容不能溢出容器，移动端需要支持基本查看和确认流程。

页面草图、视图关系和组件设想记录在 `docs/website_design_note.md`。

## 7. 预期输出

第一版应产出：

- 试卷或错题本上传记录。
- 腾讯云试卷切题结果 JSON。
- 学生人工确认的重点题记录。
- Hermes 生成的重点题总结、错因总结和复习建议。
- 周报 JSON 和周报索引文件。
- 可部署的 Hermes Web 前端。
- 比赛报告、演示脚本和演示视频素材。

## 8. 评审展示角度

面向比赛评审时，演示应强调：

- AI 智能体不是只回答问题，而是能持续组织学习材料并推动下一步行动。
- 系统承认 OCR 和批改语义识别的边界，用人工确认保证结果可信。
- 前端、存储和部署方案具备从演示阶段走向长期系统的路径。
- 输出内容以周报和行动建议呈现，便于学生和家长理解。
