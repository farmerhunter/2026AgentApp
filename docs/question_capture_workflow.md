# 试卷和错题本图像处理设计

本文档记录 Hermes 对试卷、练习册、错题本图像的处理路线。目标不是完全自动理解老师批注，而是利用腾讯云试卷切题能力完成题目定位和内容识别，再由学生或家长手动确认哪些题需要记录，最后交给 Hermes 生成错题记录、学习建议和周报。

## 1. 设计结论

第一版采用“自动切题 + 人工确认 + 智能体总结”的路线：

```text
上传试卷/错题本图像
  -> 腾讯云 QuestionSplitOCR 试卷切题
  -> Web 前端展示题目框和切题列表
  -> 学生手动选择需要记录的题
  -> 学生补充错因、得分、知识点或备注
  -> Hermes 生成重点题记录
  -> Hermes 汇总到周报 JSON
```

该方案避免依赖模型自动识别老师红笔批注、老师给分和老师改写内容。腾讯云服务负责它更擅长的题目定位、切分和文字识别；关键语义由学生或家长确认。

## 2. 适用输入

支持输入：

- 已完成的试卷照片
- 练习册页面照片
- 错题本页面照片
- PDF 或图片格式的学习材料

第一版优先支持单页图片上传。PDF、多页试卷和批量处理可以放到后续阶段。

## 3. 腾讯云能力边界

计划使用腾讯云 OCR 的 `QuestionSplitOCR` 试卷切题能力。

该能力适合：

- 识别整页中的题目区域
- 返回题目坐标
- 返回题目文字
- 返回题目选项、答案区域、图片、表格等结构信息
- 可选返回切题后的题目图片

不应假设它能可靠完成：

- 识别老师红笔对勾、叉号、圈画和划线
- 识别老师给分或扣分
- 理解老师手写批注
- 判断老师为什么扣分
- 将老师批注和题目自动绑定

因此，第一版不把“自动理解老师批改结果”作为核心能力。

## 4. 用户工作流

### 4.1 上传

用户在 Web 前端上传一张试卷或错题本图片。

上传后系统记录：

- 学生
- 科目
- 来源类型：试卷、练习册、错题本、其他
- 拍摄日期或材料日期
- 原始图片文件

### 4.2 自动切题

后端将图片提交给腾讯云 `QuestionSplitOCR`。

系统保存：

- 原始图片路径
- 腾讯云原始响应
- 每道题的坐标
- 每道题的识别文本
- 每道题的切题图片
- 识别任务状态和错误信息

### 4.3 前端确认

Web 前端展示两种视图：

1. 页面预览视图：展示原始试卷图片，并叠加题目框。
2. 题目列表视图：展示切出的题目图片、题号、识别文字和选择控件。

用户可以对每道题做标记：

- 是否记录
- 做错了
- 不确定
- 老师重点讲过
- 需要复习
- 已掌握

用户可以手动补充：

- 老师给分
- 满分
- 错因
- 知识点
- 备注
- 复习优先级

### 4.4 Hermes 总结

用户确认后，Hermes 只处理被选中的题。

Hermes 输出：

- 重点题记录
- 错因总结
- 知识点归类
- 复习建议
- 下周行动项
- 周报 JSON

## 6. 前端功能要求

第一版 Web 前端需要支持：

- 图片上传
- OCR 任务状态展示
- 原图预览
- 题目框叠加显示
- 切题列表
- 单题图片预览
- 勾选需要记录的题
- 标记错题、重点题、不确定题
- 手动填写得分、满分、知识点、错因、备注
- 保存确认结果
- 生成或查看 Hermes 总结结果

移动端需要支持拍照上传和基本题目确认；复杂批量编辑可以优先适配桌面端。

## 7. 后端功能要求

第一版后端需要支持：

- 接收上传文件
- 保存原始图片
- 调用腾讯云 `QuestionSplitOCR`
- 保存腾讯云原始响应
- 将切题结果转换为内部 JSON
- 提供前端读取切题结果的接口或静态 JSON
- 接收人工确认结果
- 调用 Hermes 总结流程
- 写入周报数据目录

## 8. 演示阶段范围

演示阶段建议只做：

1. 单页试卷或错题本图片上传。
2. 自动切题和题目列表展示。
3. 用户手动选择 1-5 道重点题。
4. 用户手动填写错因或知识点。
5. Hermes 生成学习建议。
6. 周报页面展示这些重点题和建议。

暂不承诺：

- 自动识别老师批注
- 自动读取老师给分
- 自动判断错题原因
- 多页复杂试卷批量处理
- 无人工确认的全自动错题归档

## 9. 后续增强

后续可以逐步加入：

- 多页 PDF 支持
- 批量上传
- 腾讯云试题批改 Agent 对未批改题进行辅助判断
- 多模态模型识别老师红笔批注
- 知识点自动归类
- 相似题推荐
- 错题复习状态追踪
- 错题本导出

## 10. 对比赛项目的价值

该路线符合“AI 智能体应用：我的学习生活小助手”的主题：

- AI 自动处理试卷结构，降低整理错题成本。
- 学生参与确认，避免模型误判关键学习记录。
- Hermes 将重点题转化为复习计划和周报。
- 整个流程体现智能体的感知、规划、记录和后续行动能力。
