# Textbook Summary Skill 设计

本文档定义 `textbook_summary` skill。该 skill 对应 `design_docs/03_design_brief.md` 的用例 1：上传课本 PDF，建立学习内容理解。

## 1. 目标

`textbook_summary` 的目标是将教材 PDF、讲义或已提取的教材文本转化为 Hermes 可复用的教材理解结构。

它回答：

- 这份材料属于哪个学科、年级和教材范围？
- 材料包含哪些章节、学习单元和知识点？
- 哪些知识点后续可用于关联试卷、错题、备注和周报？
- 哪些内容不确定，需要人工补充或后续校验？

## 2. 输入

第一版输入可以是：

- 教材 PDF 文件路径，演示阶段可用样例 PDF。
- 已提取的教材文本，适合在不接真实 PDF 解析时做 demo。
- 材料元数据：
  - `subject`
  - `subject_label`
  - `grade`
  - `source_title`
  - `publisher`
  - `page_range`
  - `captured_at`

## 3. 输出

输出应符合 `data/contracts/textbook_content_summary.contract.json`。

核心字段包括：

- 教材 ID。
- 学科和学科中文名。
- 章节列表。
- 学习单元列表。
- 知识点列表。
- 关键词。
- 不确定或待确认内容。

## 4. Hermes 边界

Hermes 负责：

- 提取章节和学习单元。
- 归纳知识点。
- 形成后续分析可引用的结构化摘要。
- 标注不确定内容。

Hermes 不负责：

- 第一版不保证完整解析所有 PDF 版式。
- 第一版不做版权教材全文保存。
- 第一版不处理真实学生隐私。
- 第一版不承诺自动识别所有扫描页、复杂表格和图片题。

## 5. 与 UI 的关系

对应 UI：`学习内容` 视图。

触发方式：

```text
用户选择教材 PDF
  -> 填写学科、年级、标题、页码范围
  -> 触发 textbook_summary job
  -> 生成 textbook_content_summary JSON
  -> UI 展示章节、学习单元和知识点
```

第一版可以用本地 demo 状态模拟上传和分析，但输出结构应与真实 job 保持一致。

## 6. 质量标准

- 输出必须保留学科字段。
- 章节和知识点不能凭空编造；不确定内容必须标注。
- 知识点粒度应服务后续错题和备注关联，不宜过细或过泛。
- 输出应短而结构化，避免保存长篇教材原文。

## 7. 后续扩展

- 接入真实 PDF 文本抽取。
- 支持教材版本和章节编号。
- 支持教材图片、例题和练习题摘要。
- 支持人工修订教材摘要。
- 支持按学期或单元维护学习进度。
