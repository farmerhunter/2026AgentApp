# 比赛提交 Checklist

本文用于提交前核对最终交付物。比赛要求来源见
[`design_docs/01_competition_rules.md`](../design_docs/01_competition_rules.md)。

## 1. 基本信息

- [ ] 作品名称已确定，并与技术文档、演示视频、作品链接页面保持一致。
- [ ] 参赛项目确认为“项目 2：AI 智能体应用”。
- [ ] 年度主题“我的学习生活小助手”在文档和演示中有清晰体现。
- [ ] 团队成员、学校、区县等报名信息只填写在比赛系统或私有材料中，不提交到公开仓库。

## 2. 技术文档

- [ ] 已说明项目设计背景、问题和需求。
- [ ] 已说明设计理念、目标用户和核心使用场景。
- [ ] 已说明系统架构，包括 Web UI、Hermes、数据存储、OCR/试题分割和部署方式。
- [ ] 已说明主要工作流，包括学习成果记录、材料上传、重点题确认、周报生成和学习计划建议。
- [ ] 已说明应用价值、亮点与创新之处。
- [ ] 已说明当前限制、不足和后续改进方向。
- [ ] 文档内容与 `design_docs/` 中的设计文档保持一致。

## 3. 智能体说明

- [ ] 已描述 Hermes 智能体的任务目标。
- [ ] 已描述智能体主要功能：资料整理、教材摘要、错题记录、周报生成、学习建议生成。
- [ ] 已描述智能体工作流：输入、处理步骤、工具调用、输出格式和人工确认节点。
- [ ] 已描述核心数据契约和输出文件位置。
- [ ] 已说明哪些步骤为自动执行，哪些步骤需要学生或开发者确认。
- [ ] 已说明演示阶段与长期部署阶段的差异。

## 4. 演示视频

- [ ] 视频格式为 MP4。
- [ ] 视频时长不超过 2 分钟。
- [ ] 视频展示了 Web UI 的主要入口和关键页面。
- [ ] 视频展示了至少一个材料上传或重点题确认流程。
- [ ] 视频展示了 Hermes 生成或读取周报、学习建议的结果。
- [ ] 视频中的数据全部为虚构或脱敏数据。
- [ ] 视频画面无账号、密钥、手机号、真实姓名等隐私信息。

## 5. 作品链接

- [ ] Web UI 或智能体作品已公开发布。
- [ ] 链接可在无本地开发环境的浏览器中访问。
- [ ] 链接来源安全可靠，不暴露后台管理入口、密钥或私人文件目录。
- [ ] 已准备备用说明：如演示期间网络不可用，如何用本地视频或截图说明功能。

## 6. 创作者声明

- [ ] 已准备《创作者声明》。
- [ ] 声明中的作品名称与最终提交名称一致。
- [ ] 声明中未包含不应公开的个人隐私信息。
- [ ] 签名、日期和提交格式符合比赛系统要求。

## 7. 合规与隐私

- [ ] 所有样例学生、老师、试题、成绩和备注均为虚构或脱敏。
- [ ] 未上传真实身份证号、手机号、人脸、家庭住址、学校敏感信息等隐私数据。
- [ ] 未提交真实试卷原图或未授权教材全文。
- [ ] 未在 Git 仓库中提交 API Key、云服务密钥、SSH 私钥、Cookie 或 Token。
- [ ] 公开链接不允许直接浏览私有上传目录。

## 8. 评分维度核对

- [ ] 创意维度：突出“学习生活小助手”与智能体主动规划能力。
- [ ] 技术应用维度：说明 AI、OCR/试题分割、Web UI、数据契约和部署链路如何服务功能。
- [ ] 现实价值维度：说明项目如何帮助初中生整理资料、复盘错题和形成学习计划。
- [ ] 主题思想维度：内容积极、健康，聚焦学习成长。
- [ ] 质量体验维度：演示流程清晰，UI 可用，关键功能无明显断点。

## 9. 最终打包前检查

- [ ] `deliverables/docs/` 中的文字稿已更新到最终版。
- [ ] `deliverables/video_assets/` 中的脚本和镜头清单已对齐最终视频。
- [ ] `deliverables/final_submission/` 中的最终提交信息已准备好。
- [ ] GitHub issue 中对应的交付物任务已记录完成结果和提交 commit。
- [ ] 本地、GitHub、VPS 或比赛系统中的最终版本一致。
